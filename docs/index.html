MetalSNES
SNES Emulator for macOS

A from-scratch Super Nintendo emulator built with SwiftUI, Metal GPU rendering, and a hand-written C CPU core. Vibe coded with Claude and Codex.

MetalSNES
Features

Everything from scratch

No borrowed cores, no ports. Every component was written from the ground up to understand how the SNES actually works.

C

65C816 CPU Core

Full instruction set implemented in C with a 256-entry dispatch table. Handles all addressing modes, emulation/native switching, and interrupt handling.

P

PPU Renderer

Scanline-accurate renderer supporting all 8 BG modes (0-7), sprite compositing with priority, color math, HDMA effects, and window masking.

A

APU & SPC700

Full SPC700 CPU with S-DSP: BRR sample decoding, ADSR/GAIN envelopes, pitch modulation, noise, echo buffer with FIR filtering. Real-time audio via AVAudioEngine.

M

Metal GPU Pipeline

CRT shader effects, compute-based PPU rendering, triple-buffered presentation, and integer scaling — all powered by Apple's Metal API.

D

Debug Tools

Register inspector, disassembler, memory viewer, VRAM tile browser, sprite debugger, and a live HTTP debug server on port 8765.

S

Super FX Support

Coprocessor emulation for games like Star Fox. Includes GSU instruction set, plot/color registers, and ROM/RAM banking.

Architecture

How it all connects

The emulator models each SNES subsystem as a separate module, connected through a central bus — just like real hardware.

# Build and run
xcodebuild -scheme MetalSNES -configuration Release build

# Or run a benchmark
./MetalSNES --benchmark path/to/rom.sfc

# Start the debug server
./MetalSNES --serve-rom path/to/rom.sfc
# Then visit http://localhost:8765
